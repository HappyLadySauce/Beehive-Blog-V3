# Beehive Blog v2 产品设计

## 1. 产品目标

v2 的产品目标是构建一个同时服务于创作、沉淀、展示与 AI 协作的个人知识平台。

第一阶段重点不是把所有想法都做完，而是先打通 3 条主链路：

- 内容沉淀链路
- 公开展示链路
- 搜索与 AI 辅助链路

## 2. 产品结构

v2 第一阶段建议拆成两个主要产品面：

- `Public Web`
- `Studio`

## 3. Public Web

### 3.1 产品定位

Public Web 是面向访客与读者的公开站点。

目标：

- 展示你的文章、项目、经历和作品
- 提供清晰的个人表达入口
- 提供搜索和后续知识探索入口

### 3.2 第一阶段核心页面

建议包含：

- 首页
- 文章列表页
- 文章详情页
- 项目列表页
- 项目详情页
- 个人经历 / 时间线页
- 技术专题页
- 搜索页
- 作品集页
- 关于 / 联系方式页
- 登录 / 注册页

### 3.3 首页

首页建议承担：

- 个人定位介绍
- 最新文章
- 代表项目
- 经历入口
- 技术专题入口
- 搜索入口

首页不建议做得过重，重点是建立认知路径。

### 3.4 文章页

文章体系是公开站的主内容入口。

需要支持：

- 分类、标签、专题聚合
- 推荐相关文章
- 附件与引用展示
- 评论

### 3.5 项目页

项目页应该比普通博客的“项目合集”更结构化。

建议展示：

- 项目背景
- 目标
- 技术栈
- 关键过程
- 结果
- 关联文章
- 关联作品与附件

### 3.6 经历 / 时间线页

这是 v2 差异化的重要页面。

建议以时间线为主轴展示：

- 某一阶段的经历
- 该阶段参与的项目
- 对应文章与反思
- 作品和输出

第一阶段以时间线列表和详情页为主，知识地图可后置。

### 3.7 搜索页

第一阶段搜索页目标是“好用”，不是“炫”。

建议支持：

- 关键词搜索
- 按类型过滤
- 按标签过滤
- 按时间范围过滤
- 高亮摘要

## 4. Studio

### 4.1 产品定位

Studio 是面向你的内容工作台。

目标：

- 管理所有内容实体
- 管理关系、版本和附件
- 使用 AI 辅助整理与写作
- 审阅 AI 输出
- 发布到公开站

### 4.2 第一阶段核心模块

- 仪表盘
- 内容中心
- 笔记中心
- 项目中心
- 经历中心
- 时间线事件管理
- 标签与关系管理
- 附件中心
- 搜索管理
- AI 草稿中心
- 发布管理
- 评论管理
- 用户与权限设置

### 4.3 仪表盘

建议展示：

- 最近编辑内容
- 待审草稿
- 最近发布内容
- 搜索索引状态
- 评论动态

### 4.4 内容中心

用于统一管理：

- 文章
- 笔记
- 反思 / 洞察

关键能力：

- 创建 / 编辑
- 版本对比
- 状态切换
- 发布
- 关联项目和经历

### 4.5 项目中心

用于管理项目信息与关联内容。

关键能力：

- 创建项目
- 维护项目结构
- 绑定文章、附件、经历
- 输出项目页

### 4.6 经历中心

用于按时间线组织个人经历。

关键能力：

- 创建经历阶段
- 添加时间线事件
- 关联项目、文章、作品、反思
- 形成对外展示用的经历页面

### 4.7 AI 草稿中心

这是 v2 与普通 CMS 的重要区别之一。

需要支持：

- 从内容生成摘要
- 从搜索结果生成草稿
- 从多篇笔记生成周报 / 阶段总结
- 展示引用来源
- 提交审核

### 4.8 发布管理

第一阶段需要显式区分：

- 草稿
- 待审
- 已发布
- 已归档

你作为唯一审核人，可以在这里完成：

- 审核
- 驳回
- 发布
- 归档

## 5. 关键用户流程

## 5.1 人工创作流程

```text
创建笔记 / 文章
  -> 编辑内容
  -> 绑定标签、项目、经历
  -> 保存版本
  -> 提交发布
  -> 公开站展示
```

## 5.2 AI 协作流程

```text
选择内容范围 / 搜索结果
  -> 请求 AI 生成摘要或草稿
  -> AI 输出待审稿
  -> 你审核修改
  -> 发布
```

## 5.3 经历整理流程

```text
创建时间线阶段
  -> 添加经历事件
  -> 绑定项目、文章、反思、作品
  -> 形成经历页
```

## 5.4 访客消费流程

```text
访问首页
  -> 浏览文章 / 项目 / 时间线
  -> 搜索
  -> 阅读详情
  -> 评论 / 联系
```

## 6. 简化权限在产品中的体现

第一阶段产品层面只体现以下规则：

- 公开内容对所有访客可见
- 登录内容仅对登录用户可见
- 私密内容仅 owner 可见
- AI 是否可读由单独开关控制
- 草稿和待审内容默认不对外、不对 AI 开放

这套规则应直接体现在：

- 编辑页
- 发布设置
- 搜索索引策略
- AI 调用策略

## 7. 第一阶段产品边界

### 必须完成

- 公开站内容展示
- Studio 内容管理
- 注册登录
- 评论
- 搜索
- 项目和经历管理
- AI 摘要 / 草稿待审能力

### 可以后置

- 知识地图可视化
- 复杂推荐系统
- 多人协作
- 高级审核流
- 很复杂的 Agent 运营后台

## 8. 第一阶段产品成功标准

如果 v2 第一阶段设计正确，应达到以下结果：

- 你可以稳定管理文章、笔记、项目和经历
- 访客可以清晰浏览公开内容
- 搜索能成为有用入口
- AI 能帮助你整理和起草，而不是制造混乱
- 权限边界清晰，不会把私密内容泄露出去
